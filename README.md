# CHISTOVIK

Веб-ориентированная система автоматического анализа и коррекции текста на русском языке. Проект разработан в рамках дисциплины «Системная и программная инженерия» (РТУ МИРЭА, 2026).

Система помогает улучшать качество письменных текстов за счёт проверки орфографии, оценки читаемости, анализа стиля, выявления «воды» и канцеляризмов, а также формирования рекомендаций по упрощению и улучшению формулировок.

## Функциональные возможности

- ✍️ **Ввод и редактирование текста** – удобный текстовый редактор с поддержкой русского языка.
- 🔍 **Орфографическая проверка** – подсветка ошибок и предложение вариантов исправления.
- 📊 **Оценка читаемости** – расчёт индексов (Flesch, Coleman‑Liau и др.) и их текстовая интерпретация.
- 🎭 **Классификация стиля** – определение формальности текста (формальный/нейтральный/разговорный).
- 💧 **Поиск «воды» и канцеляризмов** – подсветка избыточных выражений по встроенным спискам.
- ✂️ **Анализ сложных предложений** – выявление слишком длинных предложений и рекомендации по упрощению.
- 📋 **Панель рекомендаций** – все замечания собраны в одном месте с навигацией по тексту.
- 💾 **Сохранение истории** – возможность сохранять результаты анализа и возвращаться к ним позже.
- 🔁 **Сравнение версий** – просмотр изменений между исходным и отредактированным текстом (опционально).

## Технологический стек

### Backend
- **Python 3.10+** / **FastAPI** – высокопроизводительный веб-фреймворк.
- **SQLAlchemy** – ORM для работы с базой данных.
- **PostgreSQL** – реляционная база данных.
- **NLP-модули**:
  - [LanguageTool](https://languagetool.org/) – орфография и грамматика.
  - [textstat](https://github.com/shivam5992/textstat) – метрики читаемости.
  - [pymorphy2](https://github.com/kmike/pymorphy2/) – морфологический анализ.
  - Rule‑based модули для поиска «воды» и классификации стиля.

### Frontend
- **React.js** (или Vue.js) – клиентская часть.
- **Figma** – проектирование макетов.

### Инфраструктура
- **Docker**, **Docker Compose** – контейнеризация.
- **Nginx** – веб-сервер (в production).

## Архитектура

Проект построен по трёхуровневой клиент-серверной архитектуре:

- **Frontend** – React-приложение, взаимодействующее с API.
- **Backend** – FastAPI, обрабатывающий запросы и координирующий NLP-модули.
- **База данных** – PostgreSQL, хранящая пользователей, тексты, результаты анализов.

Все компоненты упакованы в Docker-контейнеры и могут быть развёрнуты одной командой `docker-compose up`.


## Установка и запуск

### Предварительные требования
- Docker и Docker Compose (рекомендуется)  
  *или*
- Python 3.10+, Node.js 16+, PostgreSQL 13+

### Запуск через Docker (рекомендуемый способ)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/your-org/ai-writing-assistant.git
   cd ai-writing-assistant
   ```

2. Создайте файл `.env` на основе `.env.example` и укажите необходимые переменные (например, параметры подключения к БД).

3. Запустите контейнеры:
   ```bash
   docker-compose up --build
   ```

4. Откройте приложение в браузере: [http://localhost:3000](http://localhost:3000) (или другой порт, указанный в конфигурации).

### Локальный запуск (без Docker)

#### Backend
```bash
cd backend
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows
pip install -r requirements.txt
# Настройте подключение к БД в файле .env
uvicorn main:app --reload
```
API будет доступно по адресу `http://localhost:8000`. Документация Swagger – `http://localhost:8000/docs`.

#### Frontend
```bash
cd frontend
npm install
npm start
```
Фронтенд запустится на `http://localhost:3000`.

## Использование API

Основные эндпоинты:

- `POST /analyze` – анализ текста. Тело запроса: `{ "text": "Ваш текст" }`. Возвращает JSON с подсвеченными ошибками, метриками и рекомендациями.
- `GET /history` – получение истории анализов текущего пользователя.
- `POST /save` – сохранение текущего текста в историю.

Полная спецификация API доступна в Swagger после запуска backend.
